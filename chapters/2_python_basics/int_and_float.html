2.2. Integers and Floats¶

2.2.1. Integers¶

Integers are any whole numbers (positive and negative) without a fractional part.

a=1
b=1000
c=-32
print(a,b,c)

1 1000 -32

2.2.2. Floats¶

Floats are numbers (positive and negative) that have a fractional part

d=0.1
e=0.0001
f=-0.034
print(d,e,f)

0.1 0.0001 -0.034

2.2.3. Operations¶

We can interact with both integers and floats using operators.

print(3 + 5)    # addition
print(23 - 3)   # subtraction
print(13 * 43)  # multiplication
print(12 / 2)   # division

8
20
559
6.0

For special use cases, Python gives us advanced operators.

print(17 / 3)    # classic division
print(17 // 3)   # floor division discards the fractional part
print(17 % 3)    # the % operator returns the remainder of the division
print(4**4)      # exponents

5.666666666666667
5
2
256

2.2.4. Type¶

We have so far heavily relied on 1 of Python’s special global function - print.

Let’s use another global function - type. Type returns the data type of any variable we provide as an input.

type(100)

int

type(46.85)

float

2.2.5. Casting¶

int and float are not just data types, they are also special functions themselves.

When used as a function, we can cast data from 1 type to another.

x = 80 # x is an integer
type(x)

int

y = float(x) # casting our integer(x) to a float(y)
print(y)
type(y)

80.0

float

In a few cases, when Python feels it’s appropriate, it will automatically cast data types. Such as in the following example:

# Although we start with solely integers
a = 100
b = 3
print (type(a), type(b))

<class 'int'> <class 'int'>

# Python realises it's more accurate to provide the answer as a float
answer = a/b
print(answer)
type(answer)

33.333333333333336

float

2.2.6. Python Calculations¶

One reason why Python is a very useful language is it’s ability to be a powerful and expressive calculator.

# Python easily evaluates complicated mathematical expressions
2**(32%(3+4) - (4//2)) + 47

51

# If we want to continue our calculations from where we left off
# the latest output is always conveniently saved in the '_' variable
# (much like a calculator)
_ * 100

5100
